# 上下文管理 (Context Management)

## 概念定义
在大型语言模型（LLM）驱动的编程辅助工具中，上下文管理是一项至关重要的核心能力。由于模型本身存在上下文窗口（Context Window）的限制，且编程任务往往涉及庞大的代码库、复杂的依赖关系以及长周期的交互过程，上下文管理系统负责动态地筛选、压缩和组织信息。它不仅包括对当前对话历史的截断与摘要，还涵盖了对项目结构、文件变更历史、Git 状态以及工具执行结果的实时追踪。通过高效的上下文管理，AI 助手能够克服“遗忘”问题，在处理跨文件重构、深层逻辑调试或长期项目开发时，始终保持对代码库全局状态的准确理解，从而确保生成的代码与现有架构高度一致，避免因信息丢失导致的逻辑偏差或幻觉。

## 技术细节
在 Claude Code 等现代 AI 编程工具中，上下文管理通常通过以下技术手段实现：

*   **自动压缩与摘要**：系统会定期对长对话历史进行语义压缩，提取关键决策点和代码变更摘要，以腾出上下文窗口空间给当前任务。
*   **代码库索引（Indexing）**：通过对本地代码库建立向量索引或语义映射，当模型需要特定上下文时，系统能快速检索相关文件片段，而非将整个代码库塞入 Prompt。
*   **状态持久化**：维护一个包含当前工作目录状态、Git 分支信息、待办事项列表的“工作记忆”，确保在会话中断后能够快速恢复上下文。
*   **工具调用集成**：上下文管理与 [[工具调用]] 紧密耦合，当模型执行 `Read` 或 `Bash` 操作时，返回的结果会自动被处理并注入到上下文窗口中，作为后续决策的依据。

## 应用场景
*   **大型代码库重构**：在涉及多个模块的重构任务中，系统需要同时维持对多个相关文件的引用，上下文管理确保模型不会在重构过程中丢失对依赖关系的追踪。
*   **长期项目开发**：在持续数周的开发周期中，上下文管理帮助模型记住之前的架构设计决策和已实现的 API 规范。
*   **复杂 Bug 修复**：在定位深层逻辑错误时，系统需要整合错误日志、堆栈跟踪信息以及相关的代码片段，上下文管理确保这些碎片化信息被正确关联。
